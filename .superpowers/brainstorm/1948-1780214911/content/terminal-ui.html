Funcionamento do polling

Apos disparar o workflow, o frontend consulta a API do GitHub a cada 3 segundos para obter os logs da action em execucao. Os logs sao exibidos com cores por severidade: verde (sucesso), azul (info), laranja (warning), vermelho (erro).

Tech stack do frontend

  • index.html — estrutura base, botao, container do terminal
  • app.js — logica de disparo, polling de logs, renderizacao no terminal, cores
  • style.css — tema escuro, fonte monospace, scroll automatico
  • Zero dependencias externas (vanilla JS, CSS puro)
